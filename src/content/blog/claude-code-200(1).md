# Claude Code 200% 활용하기 2편: 실전 운영법

저번 글에 이어 Claude Code를 잘 사용하는 방법을 정리해보려 합니다. 저번 편에서는 짧고 명확하게, 불필요한 말을 줄이는 것이 핵심이라고 이야기했습니다. 이번에는 제가 생각하는 실전 효율화 방법을 적어보겠습니다.

### 그래서 문제가 뭔데?

AI 에이전트가 없던 때를 떠올려보면, 개발자는 문제나 이슈가 생겼을 때 먼저 "어떻게 해결할 수 있을까?"를 고민합니다.

문제가 단순하면 빠르게 해결되지만, 복잡한 문제는 오히려 또 다른 문제를 만듭니다. 여러 사람이 각자 관점에서 문제를 보기 시작하면, 본질이 아닌 다른 지점에 포커스가 쏠리는 경우가 많습니다. 그러다 보면 방향이 멀어지기도 합니다.

그럴 때 다시 물어야 합니다.

> 그래서 문제가 뭔데?

진짜 문제가 무엇인지, 애초에 그게 문제로 정의할 수 있는 일인지까지 깊게 점검해야 합니다. 이 과정을 건너뛰면 문제 주변만 빙빙 돌다가 시간과 인력을 낭비하게 됩니다.
~~(가끔은 이런 과정 자체가 의미 있을 때도 있습니다.)~~

다시 본론으로 돌아오면, AI 에이전트에게 일을 맡길 때도 똑같습니다. 우리는 정확히 어떤 문제가 있는지 먼저 인지해야 합니다. 그래야 불필요한 말 없이 필요한 정보만 전달할 수 있습니다.

**정확한 문제 파악**에 집중합시다.

### 너 계획이 뭐니?

사람들과 일을 하다 보면 같은 대상을 봐도 해석은 다르게 나옵니다. 당연한 일이지만, 실무에서는 자주 놓치는 부분입니다.

AI도 마찬가지입니다. 같은 문제를 보고 있어도 내가 생각한 방향과 AI가 생각한 방향은 다를 수 있습니다. 보통은 이 차이를 "AI가 멍청하다"라고 표현하지만, 실제로는 정렬(alignment) 문제에 더 가깝습니다.

그래서 **"이 문제를 해결해!"**라고 바로 지시하기보다,
문제 인식이 맞는지 확인하고,
어떻게 해결할지 계획을 먼저 들어보고,
진행 중인 풀이 과정을 리뷰하는 단계가 필요합니다.

이 과정을 거쳐야 불필요한 시행착오를 줄이고, 원하는 해결점에 더 정확하게 도달할 수 있습니다.

**"너 계획이 뭐니?"**를 묻는 습관을 꼭 실천해보세요.

### 결국 에이전트도 기계다

기계에게 과한 맥락 추론을 기대하기는 어렵습니다. 사실 사람도 모호한 말에는 자주 헷갈립니다.
~~내 얘기다~~

"이거", "저거", "적당히", "아까 거" 같은 표현을 쓰면 사람도 문맥 해석에 에너지를 쓰듯, 에이전트도 마찬가지입니다.

그래서 가능하면 명확한 단어와 문장으로 요청해야 합니다. 이런 이유로 저는 상황에 따라 한글보다 영어가 더 명확하다고 느낄 때가 있습니다. 한글에는 문맥 의존적인 표현이 상대적으로 많기 때문입니다.
