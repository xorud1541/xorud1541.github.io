# Claude Code 200% 활용하기

불과 작년까지만 해도 AI 에이전트를 크게 신경 쓰지 않았는데, 지금은 생각이 완전히 바뀌었습니다. 그만큼 AI 에이전트의 발전 속도가 빠릅니다. 늦었다고 느낄 때가 가장 빠를 때라고 하니, 지금부터라도 빠르게 익숙해져 보려 합니다.

기본적으로 LLM은 `토큰(token)`이라는 최소 단위로 입력을 처리하고 결과를 생성합니다. 실제로 써보면 토큰은 무제한이 아니고, 생각보다 빠르게 소모됩니다. 그래서 결국 중요한 건 "얼마나 효율적으로 토큰을 쓰느냐"입니다.

## 토큰을 효율적으로 사용하는 기본 전략

핵심만 정리하면 아래 7가지입니다.

1. 프롬프트를 짧고 명확하게 작성하기
- 목표, 출력 형식, 제약 조건만 남기고 배경 설명은 압축합니다.
- 중복 지시는 제거합니다.

2. 긴 문서는 통째로 넣지 말고 선별하기
- 관련 구간만 발췌해 입력합니다.
- 먼저 요약본을 만든 뒤, 요약본을 기준으로 반복 작업합니다.

3. 출력 길이 제한 걸기
- `최대 길이`, `불릿 N개`, `표 1개`처럼 형식을 먼저 제한합니다.
- "핵심만 5줄"처럼 분량 제한을 명시합니다.

4. 대화 컨텍스트 정리하기
- 매 턴마다 긴 히스토리를 계속 끌고 가지 않습니다.
- 중간 결론을 짧게 요약해 컨텍스트를 대체합니다.

5. 작업을 단계별로 분할하기
- 한 번에 큰 요청을 던지기보다 단계별로 나눕니다.
- 예: 초안 생성 -> 필요한 부분만 수정

6. 반복 고정문 재사용 최소화
- 매번 긴 시스템 설명이나 규칙을 붙이기보다 템플릿을 최적화합니다.
- 자주 쓰는 지시는 짧은 키워드로 표준화합니다.

7. 고성능 모델은 필요한 구간에만 사용하기
- 분류, 전처리, 초안 단계는 저비용 모델을 우선 사용합니다.
- 최종 품질이 중요한 단계에서만 상위 모델을 사용합니다.

## 그런데 실전은 다르다

위 전략은 GPT에게 물어보면 자주 나오는 정석입니다. 다만 문제는, 이 원칙들을 매번 적용하고 습관화하는 데 시간이 꽤 걸린다는 점입니다.

예를 들어 `/init`으로 Claude의 초기 작업 규칙을 정리해 반복 업무를 줄이는 방식이 자주 언급됩니다. 분명 도움이 되는 경우가 있지만, 초기 설정 자체가 길어지면 오히려 토큰을 더 쓰는 상황도 생깁니다. 즉, 경우에 따라서는 배보다 배꼽이 더 커질 수 있습니다.

## 결론

결국 토큰 절약의 핵심은 단순합니다.

**짧게, 명확하게, 불필요한 말을 줄이는 것.**
